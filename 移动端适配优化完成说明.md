## 🎯 响应式断点说明

### Tailwind CSS 默认断点

| 断点 | 屏幕宽度 | 设备类型 |
|------|-----------|----------|
| sm | ≥ 640px | 大手机、小平板 |
| md | ≥ 768px | 平板竖屏 |
| lg | ≥ 1024px | 平板横屏、小笔记本 |
| xl | ≥ 1280px | 笔记本、小桌面 |
| 2xl | ≥ 1536px | 大桌面 |

### 本项目适配策略

1. **手机端 (< 640px)**
   - 单列布局为主
   - 横向滚动菜单
   - 大按钮便于点击
   - 隐藏次要信息

2. **平板端 (640px - 1024px)**
   - 两列布局
   - 中等字体大小
   - 适中间距

3. **桌面端 (≥ 1024px)**
   - 多列布局
   - 完整信息展示
   - 大尺寸设计元素

## 📝 移动端适配原则

### 遵循的设计原则
1. **内容优先**：重要内容优先展示
2. **简化导航**：减少层级，使用汉堡菜单
3. **大触摸目标**：按钮和链接至少44x44px
4. **横向滚动**：菜单和分类使用横向滚动
5. **字体适配**：根据屏幕大小调整字体
6. **间距优化**：移动端减小间距，节省空间
7. **图片优化**：确保图片不溢出容器
8. **表单优化**：输入框和按钮全宽显示
