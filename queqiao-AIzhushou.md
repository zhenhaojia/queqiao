# 鹊桥 - AI 婚礼帮手 Web 版需求文档

## 1. 项目概述

### 1.1 项目背景

随着婚礼行业的不断发展，新人对婚礼策划的个性化、高效化需求日益凸显。传统婚礼策划流程繁琐，涉及环节众多（如场地选择、风格设计、预算规划、嘉宾管理、流程编排等），新人往往需要耗费大量时间和精力沟通对接，且易出现信息偏差、预算超支等问题。基于此，开发 **"鹊桥 - AI 婚礼帮手 Web 版"**，借助 AI 技术为新人提供一站式、智能化的婚礼策划解决方案，简化策划流程，降低沟通成本，满足新人多样化、个性化的婚礼需求。

### 1.2 项目目标

- **打造一款集 AI 智能策划、资源对接、流程管理、预算控制于一体的 Web 端婚礼助手**，覆盖婚礼筹备全流程。
- **通过 AI 算法快速生成个性化婚礼方案**，提升策划效率，减少新人筹备时间。
- **整合婚礼行业优质资源**（场地、摄影、摄像、司仪、婚纱礼服等），为新人提供便捷的资源筛选和对接渠道。
- **实现婚礼筹备全流程的可视化管理**，帮助新人清晰掌握筹备进度，规避潜在问题。
- **建立用户信任体系**，提升用户体验和口碑，打造婚礼策划领域的优质 AI 助手品牌。

### 1.3 目标用户

- **核心用户**：22-35 岁，有婚礼筹备需求的新人，追求个性化、高效化的筹备方式，对 AI 技术接受度高，愿意通过 Web 平台简化筹备流程。
- **次要用户**：婚礼策划师，可借助 Web 版的 AI 工具快速生成方案初稿，提升工作效率；新人的父母，协助新人进行筹备管理、嘉宾确认等工作。

## 2. 核心功能需求

### 2.1 注册登录模块

- **支持手机号验证码登录、微信扫码登录、QQ 登录等多种便捷登录方式**。
- **新用户注册后需完成基础信息填写**（如姓名、婚期、所在城市、预算范围），为 AI 生成方案提供基础数据。
- **支持账号密码找回功能**，通过手机号验证码重置密码。
- **登录后自动跳转至首页**，未登录用户可浏览部分公开内容（如婚礼案例、行业资讯），但无法使用核心 AI 策划功能。
- **支持记住登录状态**（可配置有效期），减少重复登录操作。

### 2.2 AI 智能策划模块（核心）

#### 2.2.1 方案生成

- **新人输入关键需求**（婚期、所在城市、预算范围、婚礼风格：如中式、西式、森系、ins 风等、宾客人数、特殊需求：如主题元素、禁忌事项等），**AI 在 30 秒内生成个性化婚礼方案初稿**。
- **方案内容涵盖**：婚礼主题建议、场地推荐（匹配预算和风格）、流程编排（含时间节点、环节设计）、人员配置（司仪、摄影、摄像、化妆师等）、布置方案（色系、装饰元素、舞台设计草图）、预算明细（各项开支占比）。
- **支持方案一键预览**，采用图文结合的形式展示，直观清晰；支持方案页面打印功能。

#### 2.2.2 方案优化

- **新人可对生成的方案提出修改意见**（如调整风格、增减环节、修改预算分配等），**AI 根据修改意见二次优化方案**，支持多次优化直至满意。
- **支持手动编辑方案细节**，如修改流程时间、替换推荐场地、调整预算金额等。
- **支持方案保存功能**，用户可创建多个方案版本，方便对比选择。
- **支持方案分享功能**，可生成分享链接或二维码，分享至微信、QQ 等社交平台，与伴侣、家人共同讨论。

#### 2.2.3 智能问答

- **内置 AI 客服**，24 小时在线解答新人婚礼筹备过程中的各类问题（如流程疑问、合同注意事项、应急处理方案等）。
- **支持文字提问方式**，AI 快速响应并给出精准答案，复杂问题可转接人工客服。

### 2.3 资源对接模块

#### 2.3.1 资源展示

- **整合本地婚礼相关优质资源**，分类展示：场地（酒店、草坪、民宿等）、摄影摄像、司仪、化妆师、婚纱礼服、婚庆公司、花艺、甜品等。
- **每个资源展示详细信息**：图片（场地环境、作品案例）、价格区间、服务内容、用户评价、联系方式、availability（匹配婚期）。
- **支持根据婚期、所在城市、预算、风格等条件筛选资源**，精准匹配用户需求；支持列表视图与网格视图切换。
- **支持资源详情页收藏、对比功能**（最多可同时对比 3 个同类资源）。

#### 2.3.2 资源对接

- **用户可直接通过 Web 平台向资源商家发送咨询信息**，说明需求（婚期、服务类型等），商家在线回复对接；支持咨询记录历史查看。
- **支持预约到店考察或线上沟通功能**，系统记录预约信息，通过站内消息、短信（可选）发送提醒给用户和商家。
- **提供商家信誉评级和用户真实评价体系**，帮助用户筛选可靠资源，规避风险。

### 2.4 婚礼筹备管理模块

#### 2.4.1 进度管理

- **根据婚礼方案自动生成筹备进度表**，明确各阶段的核心任务（如提前 3 个月确定场地、提前 1 个月确认嘉宾名单、提前 2 周试妆等）和时间节点。
- **支持手动添加、修改任务**，设置任务优先级和截止时间。
- **通过进度条直观展示整体筹备进度**，对即将截止的任务发送站内消息提醒。
- **支持进度表导出为 Excel 或 PDF 格式**，支持日历视图展示任务安排。

#### 2.4.2 嘉宾管理

- **支持创建婚礼嘉宾名单**，录入嘉宾信息（姓名、关系、联系方式、是否携带家属、dietary 禁忌等）。
- **支持通过 Excel 模板导入嘉宾名单**，简化录入流程；支持名单批量导出。
- **支持向嘉宾发送电子邀请函**（可自定义邀请函风格、添加婚礼信息和地图导航），生成邀请函链接或二维码，通过微信、短信等方式发送。
- **实时统计嘉宾回复状态**（已确认、未确认、拒绝），自动生成出席人数统计报表，支持数据可视化展示（饼图、柱状图），方便后续场地布置、席位安排、餐饮预订。

#### 2.4.3 预算管理

- **根据 AI 生成的方案自动生成预算明细**，分类展示各项开支（场地、人员、布置、餐饮、婚纱礼服等）及占比。
- **支持手动调整各项预算金额**，系统自动计算总预算和剩余预算，当某项开支超出预设范围时发出提醒。
- **记录实际支出金额**，与预算金额对比，生成预算执行报表，支持数据可视化展示（占比图、对比图），帮助用户控制开支，避免超支。
- **支持预算表导出功能**（导出为 Excel 或 PDF），方便用户存档和分享。

#### 2.4.4 待办提醒

- **根据筹备进度表和用户添加的任务，自动推送待办提醒**，提醒方式包括站内消息、邮件（可选）、短信（可选）。
- **支持自定义提醒时间和频率**，确保用户不会遗漏重要筹备任务。

### 2.5 内容展示模块

- **婚礼案例库**：展示不同风格、不同预算的真实婚礼案例，包含详细的方案介绍、现场图片/视频、费用明细，为用户提供参考；支持案例搜索、筛选、收藏。
- **行业资讯**：推送婚礼筹备攻略、最新婚礼趋势、避坑指南、法律常识（如婚前财产、婚礼合同注意事项）等内容；支持资讯分类、标签筛选、收藏、分享。
- **灵感库**：分类展示婚礼主题灵感、布置元素、婚纱造型、伴手礼推荐等内容，支持用户收藏喜欢的内容，用于后续方案参考；支持按风格、季节等维度筛选。

### 2.6 个人中心模块

- **个人信息管理**：可修改姓名、手机号、婚期等基础信息，上传头像。
- **我的方案**：查看、编辑、删除已保存的婚礼方案；支持方案搜索、排序（按创建时间、修改时间）。
- **我的预约**：查看向资源商家发送的预约记录，包括预约状态、商家信息、预约时间等；支持预约筛选（按状态、时间）。
- **我的收藏**：查看收藏的案例、灵感、资讯等内容；支持分类查看、批量取消收藏。
- **消息中心**：查看系统通知、商家回复、任务提醒等消息；支持消息已读/未读标记、批量删除。
- **设置**：包含消息通知设置（站内、邮件、短信）、隐私政策、用户协议、意见反馈、关于我们等功能。
- **退出登录**：支持一键退出当前账号。

## 3. 非功能需求

### 3.1 性能需求

- **响应速度**：页面首次加载时间≤3 秒，二次加载时间≤1.5 秒，AI 生成方案时间≤30 秒，智能问答响应时间≤5 秒。
- **稳定性**：在正常使用情况下，无页面崩溃、卡顿现象，连续运行 24 小时无异常。
- **并发能力**：支持至少 20000 名用户同时在线使用，核心功能（如方案生成、资源筛选）在高并发场景下仍能正常响应。
- **存储能力**：支持用户上传图片、视频等内容，单个用户存储容量≥100MB，可根据需求扩展。

### 3.2 兼容性需求

- **支持主流浏览器**：Chrome（最新版本及前两个版本）、Firefox（最新版本及前两个版本）、Safari（最新版本及前两个版本）、Edge（最新版本及前两个版本）。
- **适配不同屏幕尺寸**：PC 端（最小分辨率 1366×768）、平板设备（横屏模式），保证界面布局合理、显示清晰，操作便捷。
- **兼容不同网络环境**（4G、5G、Wi-Fi），在弱网络环境下（网络速度≤1Mbps），核心功能仍能正常使用，非核心功能可提示用户网络不佳。

### 3.3 安全需求

- **用户信息安全**：对用户的个人信息（姓名、手机号、婚期等）进行加密存储，严格遵守隐私政策，不泄露用户信息。
- **账号安全**：采用短信验证码验证、密码加密等方式保障账号安全，防止账号被盗用；支持登录异常提醒（如异地登录）、账号锁定功能。
- **数据安全**：定期对用户数据进行备份，防止数据丢失；保障数据传输过程中的安全，采用 HTTPS 协议加密传输。
- **支付安全**（若后续增加支付功能）：接入正规第三方支付平台，保障用户支付过程的安全，防止支付风险。

### 3.4 易用性需求

- **界面设计**：简洁、美观、直观，符合 Web 端操作习惯，色彩搭配温馨（契合婚礼主题），避免复杂的操作逻辑。
- **操作流程**：核心功能操作步骤≤3 步，用户无需学习即可快速上手；提供操作引导（如新手引导弹窗、功能提示 tooltip），帮助新用户熟悉平台使用。
- **反馈机制**：操作结果及时反馈（如提交成功、修改成功、提醒信息等），让用户清晰了解操作状态。
- **容错性**：用户输入错误信息时（如手机号格式错误、日期选择不合理），系统及时给出明确的错误提示，并引导用户正确输入。
- **导航设计**：清晰的主导航和次级导航，支持面包屑导航，方便用户了解当前页面位置及跳转。

## 4. 页面原型与交互要求

### 4.1 原型要求

产品原型需覆盖所有核心功能模块，明确各页面的布局、元素位置、跳转逻辑，确保页面之间的连贯性和一致性。原型需包含首页、注册登录页、AI 方案生成页、方案详情页、资源列表页、资源详情页、筹备进度页、嘉宾管理页、预算管理页、个人中心页等关键页面。

### 4.2 交互要求

- **页面跳转流畅**，无卡顿、延迟现象，跳转逻辑清晰，符合用户的操作习惯；支持页面前进/后退操作。
- **按钮、输入框等交互元素的反馈及时、明确**，如按钮点击时有 hover 效果和点击效果，输入完成后有确认提示。
- **弹出层（如提醒弹窗、编辑弹窗）的出现和消失动画自然**，不影响用户视线和操作；支持点击空白处关闭非关键弹窗。
- **支持下拉刷新、上拉加载更多功能**（如资源列表页、案例库页），支持列表排序、筛选操作的无刷新更新。
- **支持键盘快捷键操作**（如 Ctrl+S 保存方案、Esc 关闭弹窗等），提升操作效率。
- **表单提交支持即时验证**，减少用户操作失误；支持表单数据本地暂存，防止意外刷新导致数据丢失。
